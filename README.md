# EPUB → BBCode TXT 转换插件

## 一、概述

本插件基于Sigil插件机制开发，可将EPUB电子书中的XHTML内容转换为格式规整的BBCode纯文本文件。输出文件可直接用于论坛、博客等支持BBCode的环境。插件提供图形化配置界面，支持自定义制作信息、图床链接批量导入、自定义替换规则等高级功能。

## 二、主要功能

|功能模块|说明|
|-|-|
|**书名自动提取**|从OPF文件中读取`<dc:title>`，作为输出文本第一行。|
|**制作信息模板**|用户可编辑文本模板，支持`[find]`标记自动查找正文信息（如作者），模板可自定义。|
|**内容简介处理**|自动将内容简介区域用`[center]`标签包裹，上下各加一个空行。|
|**图床快速导入**|列出EPUB内所有图片，显示缩略图，支持批量粘贴图床链接、勾选删除图片，输出`[img]URL[/img]`。|
|**脚注内联转换**|自动识别脚注标记，提取注释内容并替换为`（注释文本）`，删除原注释区块。|
|**Ruby注音处理**|将`<ruby>正文<rt>注音</rt></ruby>`转换为`[ruby=注音]正文[/ruby]`，支持多`<rt>`处理。|
|**样式BBCode化**|加粗(`<b>`) → `[b]`，斜体(`<i>`) → `[i]`，删除线(`<s>`) → `[s]`，水平线(`<hr/>`) → `[hr]`，还原水平对齐方式(`text-align`) → `[left][center][right]`。|
|**分隔符美化**|识别`class="start-6em"`并转换为`[center][b]……[/b][/center]`。|
|**标题层级保留**|解析导航文件（NCX/NAV），根据Spine顺序匹配正文标题，输出`[center][b]标题[/b][/center]`。支持「强制注入」未匹配标题。|
|**自定义替换规则**|支持加载`rule.json`文件，按顺序执行正则替换（如全角转半角、标点修正等）。|
|**智能换行控制**|图片和标题上下各保留一个空行；普通段落间单换行；删除仅由空格/全角空格组成的行。|
|**特殊处理指定页面**|可配置删除指定页面或自动添加整页`[center]`居中标签。|

## 三、处理逻辑（工作流程）

1. **预扫描阶段**

   * 解析OPF文件，定位导航文件（NCX 或 NAV）。
   * 提取导航条目（标题 + 目标文件 + 锚点），构建标题池，并记录每个标题在Spine中的索引（防回溯）。
   * 扫描全书，收集所有脚注内容，建立全局脚注映射表。
2. **逐页转换**  
对每个页面依次执行以下操作：

   * **清理基础结构**：删除标签间空行、空标签。
   * **脚注处理**：定位脚注链接与注释定义块，将注释内容内联替换原链接，并删除注释区块。
   * **Ruby 转换**：提取注音和正文，生成`[ruby=...]...[/ruby]`。
   * **图片处理**：将`<img>`替换为`[img]URL[/img]`，根据用户勾选决定删除或使用图床链接。
   * **样式转换**：将特定class或标签转为 BBCode。
   * **分隔符保护**：将`class="start-6em"`临时标记为`[segmentation]`。
   * **剥离标签**：删除所有剩余HTML标签，将块级标签转换为换行。
   * **标题匹配**：根据当前页文件名和Spine索引，从标题池中取出期望标题，与正文行进行比对（去除空格后完全相等）。匹配成功则输出`[SYS_TITLE]标题[/SYS_TITLE]`。
   * **强制注入**：若用户开启「注入标题」开关，且某预期标题未匹配，则在页面顶部强制插入`[SYS_TITLE]`。
3. **全局排版**

   * 合并所有页面内容，加上制作信息头部。
   * 应用`rule.json`中的自定义正则替换。
   * 删除仅由空格/全角空格组成的行。
   * 统一处理换行：图片和标题上下各保留一个空行（`\n\n`），普通段落间单换行。
   * 将`[segmentation]`还原为`[center][b]...[/b][/center]`。
   * 将`[SYS_TITLE]`替换为`[center][b]标题[/b][/center]`，并添加换行。
4. **输出保存**

   * 弹出文件保存对话框，默认文件名取自书名（已过滤非法字符）。
   * 用户可自定义导出路径（插件顶部`DEFAULT_EXPORT_PATH`可预设固定目录）。
   * 写入UTF-8编码的TXT文件。

## 四、使用方法

### 1\. 安装插件

* 在菜单栏`插件` → `编辑插件`中添加插件。

### 2\. 运行插件

* 在Sigil中打开支持的EPUB文件。
* 点击菜单栏`插件` → `EPUB2BBCode`。

### 3\. 步骤一：制作信息配置

* **第一行**为书名（自动从OPF读取，可修改）。
* 使用`[find]查找词[/find]`标记：插件会在全文中搜索该词后面的文本（直到换行），并自动填入。例如`[find]作者：[/find]`会提取作者信息。
* **內容簡介**区域会被自动包裹`[center]`标签。
* 可点击**保存模板**将当前配置保存到插件目录（`bbcode_config.json`）。
* 点击**编辑rule文件**可自定义替换规则（见下文）。
* **转换分割线**（`<hr>`）勾选后，将`<hr>`转换为`[hr]`，否则删除。
* 点击**查找**执行`[find]`标记替换。
* 点击**下一步**进入图床配置。

### 4\. 步骤二：图床导入

* 插件会列出EPUB内所有图片，显示缩略图、文件名。
* **左侧删除框**：勾选后图片会被删除（不输出）。
* **右侧文本框**：可粘贴图床URL（支持批量粘贴）。
* **批量添加**：将剪贴板中的多行链接按顺序分配给图片（会弹出确认对话框）。
* **清除链接**：清空所有图片的 URL（二次确认）。
* **注入标题**：勾选后，若某页预期标题未在正文中匹配到，会在该页顶部强制插入标题（防止漏标）。
* 点击**确认转换**开始处理。

### 5\. 导出结果

* 转换完成后弹出保存对话框，默认文件名`书名.txt`。
* 选择保存路径后，插件生成最终BBCode文本文件。

## 五、自定义替换规则（rule.json）

插件支持通过`rule.json`文件自定义文本替换规则。用户可在插件目录下创建或编辑该文件，定义多组正则替换规则。规则按文件中定义的顺序依次执行，可实现全角转半角、标点符号修正、特定字符串替换等操作。具体格式请参考插件内置的示例文件或查阅开发文档。

## 六、注意事项

* **适用工具**：Sigil 1.9.0+。
* **适用对象**：以[《台灣 EPUB 3 製作指引》](https://github.com/dpublishing/epub3guide)为标准，轻小说为主的EPUB电子书。
* **仅支持EPUB3和EPUB2**（需包含NCX或NAV导航文件）。
* **标题匹配规则**：去除所有空格后完全相等。若正文标题含额外标点（如「一。」 vs 「一」）或是图片，可能导致匹配失败，此时可开启「注入标题」开关强制插入。
* **配置文件**：`bbcode_config.json`和`rule.json`保存在插件目录下，卸载插件时会自动删除，请注意备份。
* **日志输出**：转换过程中控制台会输出详细日志，若遇错误可查看插件目录下的`error_log.txt`。

## 七、常见问题

|问题|解决方法|
|-|-|
|标题未被识别|检查导航文件是否规范；开启「注入标题」开关强制插入；查看日志中「标题池」是否正确加载。|
|脚注未内联|确保脚注使用`<a href="#id">`且对应`<li id="id">`或`<p id="id">`；检查日志中是否有「匹配-注释」输出。|
|图片不显示|确认图床链接有效；若使用本地文件名，输出为`[img]文件名[/img]`，需自行上传或忽略。|
|字符替换异常|插件不会自动处理字符，若仍有问题，请检查`rule.json`是否有误。|
